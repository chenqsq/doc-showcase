# AI主导学习平台-总体架构设计

> 文档层级：平台层  
> 文档目的：说明平台层如何承接学生主线、后台演化和智能体群协同  
> 核心结论：架构重点不是分多少层，而是保证学习地图、闯关学习、画像回流、笔记沉淀和知识进化能在同一套单机方案里稳定闭环

## 1. 平台结构

| 层级 | 负责什么 |
| --- | --- |
| 学生前端 | 选科、地图、闯关、笔记和成长展示 |
| 后台前端 | 资料注入、知识演化、策略分析和审计 |
| 平台服务层 | 会话、地图、画像、日志和任务编排 |
| AI教师智能体群引擎 | 诊断、重规划、讲解、评分、笔记生成、资料入库 |
| 数据与资源层 | PostgreSQL、文件存储、知识资产 |

## 2. 页面承接

平台固定承接 6 个页面：

1. 选科与开学页
2. AI学习地图页
3. AI闯关学习页
4. 笔记复习与成长页
5. 资料注入与知识库演化后台
6. 系统自治与策略分析后台

## 3. 主数据流

`科目选择记录 -> 学习启动会话 -> AI学习地图 -> 当前关卡 -> 成长反馈事件 -> 学习画像 -> 复习笔记包 -> 知识注入任务 -> 知识演化记录`

## 4. 单机部署约束

- 前端静态站点与 Python 单体分离部署
- PostgreSQL 承接主数据和审计日志
- COS / MinIO 承接资料和思维导图资源
- ADP 和 SSE 承接多 Agent 工作流与流式输出
