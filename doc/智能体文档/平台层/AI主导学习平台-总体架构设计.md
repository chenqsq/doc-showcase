## 一句话先记住

> 架构的关键不是“画几层”，而是保证课堂资料上传、知识重构、学生伴学、错题复练、教师洞察这 5 条链路在同一套对象和同一套部署方案里稳定闭环。

## 1. 双层包装原则

### 1.1 对外作品定义

对外统一作品名：

`知脉课堂：面向高校课堂的全模态知识重构与自适应伴学智能体`

对外叙事重点：

- 面向高校课堂真实教学场景
- 支持资料上传后的自动知识重构
- 前端优先展示学生、教师、管理三类工作台
- 兼顾评委演示、访问体验与后续单机部署落地

### 1.2 对内工程定义

内部架构真源继续保留：

- AI主导学习平台：平台编排、数据沉淀、访问控制、页面承接
- AI教师子引擎：知识重构、伴学对话、错题归因、教师洞察生成
- 高等数学示范：第一门完整示范学科，用于验证扩科模板

## 2. 分层架构

| 层级 | 负责什么 | 不负责什么 |
| --- | --- | --- |
| 展示与接入层 | Vue 前端工作台、角色路由、REST/SSE 接入、评委访问体验 | 不直接承载模型编排 |
| 平台编排层 | 会话管理、任务编排、对象持久化、权限控制、日志审计 | 不直接替代学科策略 |
| AI教师子引擎层 | 知识重构、路径规划、伴学对话、评分归因、教师洞察 | 不单独维护业务主数据 |
| 学科与知识层 | 高数目录、先修关系、题型模板、课堂知识资产 | 不重造平台能力 |
| 交付与运营层 | 比赛材料、视频脚本、答辩口径、访问手册 | 不定义平台底层对象 |

## 3. 核心对象主链

平台正式主链如下：

`课堂素材包 -> 知识重构结果 -> 学习会话 -> 任务卡 -> 掌握度快照 -> 错题画像 -> 教师洞察摘要`

补充沉淀对象：

- 课节笔记
- 个人总复习本
- 学习记录
- 访问凭证

这条链路决定了平台不是单轮问答，而是从课堂输入一直走到学生个体学习和教师群体洞察。

## 4. 5 条正式业务链路

### 4.1 课堂资料上传

- 教师或管理员上传课件、讲义、题单、课堂图片、录音转写结果
- 平台生成课堂素材包并做格式校验、来源记录、状态追踪

### 4.2 知识重构

- 子引擎抽取知识点、例题、误区、先修关系
- 输出知识重构结果，供前端页面和后续伴学路径直接使用

### 4.3 学生伴学

- 学生进入学习会话
- 平台按任务卡驱动子引擎流式输出讲解、练习、追问与提示

### 4.4 错题复练

- 平台根据掌握度快照和错题画像生成变式题、补桥题、复习任务
- 支持间隔复习调度和多轮回流

### 4.5 教师学情洞察

- 教师查看班级风险分布、知识点热区、停滞链路与干预建议
- 聚合视图来自教师洞察摘要，不直接堆原始对话

## 5. 前端优先的页面承接

比赛版前端固定为 6 个核心页面，不做营销首页：

1. 课堂工作台首页
2. 课堂资料上传与重构页
3. 学生伴学会话页
4. 错题与复习页
5. 教师洞察页
6. 管理与接入配置页

页面层要求：

- 第一屏就是可操作工作台，不走宣传式落地页
- 每页都能映射到至少 1 个评审关注点
- 每页都要能落到统一对象契约和正式 API

## 6. 技术与部署口径

### 6.1 前端

- Vue 3
- TypeScript
- Vite
- Vue Router
- Pinia
- Naive UI
- Tailwind CSS
- VueUse Motion
- ECharts

### 6.2 后端

- Go 1.24
- Gin
- pgx / sqlc
- PostgreSQL 16
- 腾讯 ADP
- HTTP SSE V2

### 6.3 文件与部署

- 腾讯 COS 为默认对象存储
- 本地开发兼容 MinIO
- 单机部署采用 Caddy 或 Nginx 反向代理
- systemd 守护进程、健康检查、优雅重启
- PostgreSQL 备份恢复
- ADP 不可用时提供降级输出

## 7. 单机高可用原则

本项目当前不采用微服务架构。正式口径如下：

- 单机服务优先，追求比赛周期内可实现、可维护、可演示
- 不把 Redis、MQ、微服务拆分写成 v1 前置条件
- 将高可用收敛为单机级别的守护、监控、重启、备份与降级

## 8. 架构边界

### 8.1 平台层必须长期稳定

- 统一对象契约
- API 分组
- 会话与任务编排
- 访问控制与日志
- 页面承接与访问说明

### 8.2 子引擎层允许持续增强

- 算法优化
- 提示词与流程编排
- 模型替换
- 学科策略增强

### 8.3 学科层允许模板化扩展

- 高数不是唯一学科边界
- 扩科时补充目录、先修图谱、题型模板与知识资产

## 读完后你应该带走什么

- 比赛包装和工程真源已经统一，不再冲突。
- 平台的核心是 5 条链路和 7 个正式对象，而不是某个单点页面。
- 当前最佳落地方案是 Vue 3 前端加 Go 单体后端的单机高可用架构。
