# OCR与资料电子化负责人-职责与执行手册

> 文档层级：平台层  
> 文档目的：把教材、试卷、讲义等原始资料如何转成可检索知识资产写成可执行手册，方便直接分配给知识库负责人落地  
> 核心结论：OCR 负责人不只是“把纸拍清楚”，而是要把原始资料变成结构化、可检索、可抽样验收、能支撑 Agent 稳定回答的知识资产  
> 目标读者：OCR 负责人、知识库实施者、项目负责人、工作流负责人

## 与其他文档的边界

本文只负责回答：

- 没有电子版的教材和试卷怎么处理
- OCR 完成后怎样校对和拆分
- 什么样的内容才能算正式知识资产
- OCR 负责人最终到底交付什么

本文不替代平台字段真源，也不替代 Agent 工作流手册。

## 1. 角色定位

OCR 与资料电子化负责人要负责把原始教学资料走完下面这条链路：

`纸质资料 / 图片 / PDF -> OCR 初稿 -> 人工校对 -> 结构化拆卡 -> 元数据标注 -> 批次入库 -> 抽样验收`

如果只做到 OCR 文本而没有拆卡、打标签和验收，这份资料还不能算完成交付。

## 2. 你负责什么

1. 维护原始素材台账。
2. 制定每一批资料的扫描与命名规则。
3. 做 OCR 初稿并完成人工校对。
4. 把资料拆成知识点卡、例题卡、试卷题目卡、课堂重构笔记和错题卡。
5. 给每份知识资产补齐最低元数据字段。
6. 输出可供检索与联调使用的正式批次。

## 3. 每轮怎么做

### 3.1 第一步：登记原始素材

每拿到一批资料，先建台账，至少记录：

- 来源
- 资料类型
- 学科与章节
- 是否纸质
- 是否已有电子版
- 当前状态
- 负责人
- 预计入库批次

素材台账是后面追踪缺页、漏题、重复入库和版本回退的唯一依据，不能省。

### 3.2 第二步：扫描或拍照

扫描或拍照时至少满足：

- 优先使用平板扫描或手机扫描模式，避免普通相机直拍造成透视变形。
- 文字页清晰度尽量达到 `300 dpi` 级别。
- 保证页面完整，不裁掉页码、题号、图像和公式边缘。
- 阴影、折痕、斜拍严重的页面必须重拍。
- 图像题、几何题、函数图像题单独复检。

### 3.3 第三步：生成 OCR 初稿

OCR 初稿只是一份中间产物，重点是把下面几类错误找出来：

- 公式被识别成乱码
- 题号顺序丢失
- 图片说明漏掉
- 上下标、分式、根号错位
- 表格列错位

### 3.4 第四步：人工校对

人工校对必须优先盯下面 5 类内容：

1. 公式
2. 题号
3. 图表与图注
4. 条件与结论的边界
5. 正确答案与解析对应关系

如果一页里有公式密集区或多图混排，宁可慢一点，也不要直接把 OCR 初稿当正式文本。

### 3.5 第五步：按知识资产拆卡

拆卡时不能整章整份直接入库，必须按用途拆到可检索颗粒度：

| 原始资料 | 正式拆分方式 | 最终产物 |
| --- | --- | --- |
| 教材 | 按知识点、定义、定理、例题拆开 | 知识点卡、例题卡 |
| 试卷 | 按题目或题组拆开 | 试卷题目卡、标准答案卡、错因卡 |
| 讲义 / PPT | 按课时主题拆开 | 课堂重构笔记、讲义摘要 |
| 学生错题 | 按错因拆开 | 错题卡、误区卡 |

拆卡的目标不是“好看”，而是让 Agent 在检索时能尽快命中最小有用片段。

### 3.6 第六步：补元数据与标签

每张卡最低要补齐下面字段：

- `subject_category`
- `course_id`
- `module_id`
- `module_label`
- `chapter_id`
- `chapter_label`
- `resource_type`
- `role`
- `source_type`
- `source_ref`
- `keywords`
- `aliases`
- `difficulty`
- `status`

同时要保证：

- 标题能让人一眼看出主题。
- `source_ref` 能追溯到原教材页码或试卷来源。
- `keywords` 与 `aliases` 覆盖学生可能使用的通俗说法。

### 3.7 第七步：形成批次入库包

每一批入库包至少包含：

1. 原始素材清单
2. OCR 校对稿
3. 结构化知识资产
4. 元数据与标签表
5. 本批次说明
6. 抽样问答记录

## 4. 你的输入是什么

你需要拿到：

- 项目负责人给出的课程范围和优先级
- 章节树与模块命名规则
- 元数据字段和命名规范
- 当前要支撑的演示场景

## 5. 你的输出是什么

你必须交出 6 类成果：

1. 素材台账
2. OCR 校对稿
3. 结构化知识资产
4. 标签规范表
5. 入库批次记录
6. 抽样验收记录

## 6. 你交接给谁

### 6.1 交给工作流负责人

- 可检索的正式知识资产
- 标签和检索字段说明
- 学生问答、教师运营、课堂重构三类场景分别该检什么

### 6.2 交给项目负责人

- 当前批次完成度
- 还缺哪些模块
- 哪些资料还不能用于公开演示

## 7. 验收标准

OCR 与资料电子化工作完成时，至少要满足：

1. 任意一张知识卡都能追溯回原始资料。
2. 教材、试卷、讲义都已经拆到可检索颗粒度。
3. 公式、题号和图表经过人工校对。
4. 元数据字段完整，没有明显空缺。
5. 抽样问答能稳定召回对应资料。
6. 批次说明能让别人接手继续做，不依赖口头记忆。

## 8. 常见失误

- 把整章 PDF 直接扔进知识库，没有拆卡。
- 只修正文案，不修公式和题号。
- 题目和解析拆散后没有建立对应关系。
- 资料标题随手写，后面无法检索。
- 批次做完没有记录来源和验收结果，后续无法追责或回查。
