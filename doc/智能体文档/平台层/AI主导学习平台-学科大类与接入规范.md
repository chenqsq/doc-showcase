# AI主导学习平台-学科大类与接入规范

> 文档层级：平台层  
> 文档目的：定义学科接入与扩科为什么是平台能力主线，以及平台如何用统一规范承接多学科扩展  
> 核心结论：扩科不是新增一个“学科设计者角色”，而是平台在统一对象契约、统一模板和统一槽位下持续扩展学科能力的主线  
> 目标读者：产品负责人、扩科实施者、研发协作者、答辩准备者  
> 上游真源：[AI主导学习平台-角色主线与阶段地图.md](./AI主导学习平台-角色主线与阶段地图.md)、[AI主导学习平台-统一对象与接口契约.md](./AI主导学习平台-统一对象与接口契约.md)  
> 下游文档：[学科接入模板.md](../学科层/学科接入模板.md)、[高等数学-平台接入示范.md](../学科层/高等数学-平台接入示范.md)  
> 适用范围：学科大类规划、扩科槽位、学科接入边界、平台扩展口径

## 与其他文档的边界

本文定义的是“平台怎样扩科”。  
学科对象字段的正式定义回到统一对象契约文档；某一学科的实际填写样例回到学科层文档。

## 一句话先记住

> 学科接入与扩科不是独立角色主线，而是平台主线之一：平台先把公共对象、字段和槽位定住，再让每门学科往统一模板里补内容。

## 1. 为什么扩科是平台能力主线

如果平台只能服务高等数学，它最多是单学科作品。  
平台之所以成立，是因为它能把下面这些公共能力复用到更多学科：

- 学习档案与学习会话
- 当前任务卡与推进/回补
- 子引擎回流结果与双层笔记
- 教师运营摘要与风险干预入口
- 产品接入与学习记录沉淀

## 2. 平台固定学科大类

| 学科大类 | 典型学科 | 为什么适合平台化管理 |
| --- | --- | --- |
| `数学` | 高等数学、线性代数、概率统计 | 概念递进清晰，补桥需求强 |
| `语言` | 大学英语、日语、写作表达 | 长期积累明显，适合双层笔记与阶段复习 |
| `计算机/专业技能` | 程序设计、数据结构、办公技能 | 兼具概念理解与任务训练 |
| `考试/证书` | 四六级、计算机等级考试、教师资格证 | 目标明确，适合阶段推进和专项复习 |

## 3. 学科接入与扩科主线

### 图 1：扩科主线

```mermaid
flowchart LR
    A[统一对象契约] --> B[学科接入模板]
    B --> C[学科目录与补桥逻辑]
    C --> D[学科示范]
    D --> E[多学科扩展]
```

扩科主线固定包含 4 个步骤：

1. 平台先给出统一对象与统一字段
2. 学科接入模板承接这些槽位
3. 某门学科按模板补目录、补桥、策略、资源
4. 学科示范证明接入成立，再复制到更多学科

## 4. 接入规范

每门新学科接入平台时，至少要补齐下面这些项：

| 接入项 | 说明 |
| --- | --- |
| 学科大类 | 该学科属于哪个一级大类 |
| 学科定位 | 入门课、主线课、训练课或考证课 |
| 目标人群 | 至少 2 类真实人群 |
| 目录结构 | 至少补齐 `阶段 -> 模块 -> 课节` |
| 补桥逻辑 | 明确卡点、回补条件、回补节点 |
| 专属策略 | 该学科特有的讲解、练习、评估方式 |
| 笔记与评估模板 | 对齐课节笔记、个人总复习本、教师运营摘要 |
| 接入字段槽位 | 说明如何承接 `visitor_biz_id`、`custom_variables`、`chapter_id`、`role` |

## 5. 权限边界与占位结构

本文是 `权限边界与占位结构` 的唯一正式定义源。  
它的意思是：平台先把公共机制和扩科槽位固定下来，新学科只往这些槽位里补学科资产，而不是重写平台。

| 类别 | 平台固定 | 学科可补 | 说明 |
| --- | --- | --- | --- |
| 公共对象 | 学习档案、学习会话、当前任务卡、子引擎回流结果、双层笔记、教师运营摘要 | 不可改写 | 统一对象契约优先 |
| 平台主机制 | 建档、推进、回补、阶段复习、教师运营入口、产品接入接口 | 不可改写 | 新学科复用，不重造 |
| 学科资产槽位 | 目录、补桥逻辑、资源偏好、模板资产 | 可补齐 | 这是学科真正需要填写的部分 |
| 扩科槽位 | 新学科示范、新学科大类样例 | 可逐步新增 | 这是平台未来增长空间 |

## 6. 第一门示范学科的意义

当前第一门完整示范学科固定为 `数学 -> 高等数学`。  
它要同时证明 3 件事：

- 学生主线在学科层能跑通
- 教师主线在学科层有摘要与干预入口
- 扩科主线确实能由统一模板承接

## 7. 扩科时不该做什么

- 不把某门学科写成平台总纲
- 不把扩科改写成新增平台角色
- 不让学科配置手册变成平台默认配置
- 不让扩科时重做任务卡、学习会话、教师运营摘要等公共对象

## 读完后你应该带走什么

- 扩科是平台能力主线，不是独立角色。
- 统一对象契约和统一模板，是扩科的前提。
- 高等数学是第一示范，不是扩科的唯一终点。
