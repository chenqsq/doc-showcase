# AI主导学习平台-统一对象与接口契约

> 文档层级：平台层  
> 文档目的：定义平台长期稳定依赖的核心对象与接口分组  
> 核心结论：平台要长期成立，必须让学习地图、重规划、画像、笔记和知识进化沿同一套对象流转

## 1. 核心对象

| 对象 | 作用 | 去向 |
| --- | --- | --- |
| `科目选择记录` | 记录学生当前选择的科目与优先级 | 选科与开学页 |
| `学习启动会话` | 承接一次开学或续学入口 | 选科与开学页、AI学习地图页 |
| `AI学习地图` | 承接某门课的动态学习结构 | AI学习地图页 |
| `地图节点` | 承接主线、补桥、复习、挑战、Boss、奖励等节点 | AI学习地图页、AI闯关学习页 |
| `重规划事件` | 记录地图为什么被改、怎么改、何时回主线 | AI学习地图页、后台日志 |
| `闯关学习任务` | 承接当前节点的目标和通过条件 | AI闯关学习页 |
| `学习画像` | 承接掌握度、薄弱基础、错误模式和节奏偏好 | 笔记复习与成长页、后台 |
| `成长反馈事件` | 承接通关反馈、能力变化、地图推进和解锁 | AI闯关学习页 |
| `复习笔记包` | 承接思维导图、结构化笔记和复习计划 | 笔记复习与成长页 |
| `知识资产包` | 承接资料入库后的结构化知识资产 | 资料注入后台 |
| `知识演化记录` | 承接资产变化如何影响地图和策略 | 资料注入后台、系统自治后台 |
| `策略快照` | 承接地图规划和反馈策略版本 | 系统自治后台 |

## 2. 接口分组

1. 选科与开学
2. 学习地图生成与短诊断
3. 实时地图状态与重规划事件
4. 闯关学习会话
5. 作答评分与画像更新
6. 笔记 / 思维导图 / 复习计划
7. 资料注入与自动入库
8. 系统自治与策略分析

## 3. 契约原则

- 地图调整必须有显式 `重规划事件`
- 画像更新必须能回链到具体学习会话
- 笔记包必须能按单关、一轮、阶段检索
- 演化记录必须能回链到具体资料任务
- 前端优先消费结构化对象，不消费拼接型大文本
