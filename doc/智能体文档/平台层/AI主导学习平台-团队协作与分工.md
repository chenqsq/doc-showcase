# AI主导学习平台-团队协作与分工

## 与其他文档的边界

本文只负责回答 4 个问题：

1. 这一轮开发为什么要按三人协作而不是各做各的
2. 三个岗位分别负责什么、交接给谁
3. 一期内 `P0 / P1 / P2` 怎么并行推进
4. 最终怎样一起收口到 `master` 和 GitHub Pages

本文不替代知识库规范、提示词模板、工作流联调细节，也不替代平台对象字段真源。

## 一句话先记住

> 一期的真实目标不是拆成三个互相等待的小项目，而是让分工、知识库和 Agent 工作流在同一轮开发里闭环，并最终落到同一个公开站点。

## 1. 一期内并行推进的三条工作线

| 工作线 | 正式定位 | 这一轮必须看到什么结果 |
| --- | --- | --- |
| `P0` | 学生主闭环稳定线 | 学生能完成诊断、讲解、练习、复盘这一条最小学习闭环 |
| `P1` | 教师侧与展示增强线 | 教师能看到学生状态、错因摘要、补讲建议和演示结果 |
| `P2` | 工作流接入与产品化联调线 | 多 Agent 路由、变量透传、检索绑定、回归样例和发布链路能稳定联通 |

统一口径如下：

- `P0 / P1 / P2` 继续保留，但现在代表能力分层，不再代表严格串行阶段。
- 三条线可以有依赖，但必须在同一期内一起收口。
- 首页的三个入口就是这三条线最常用的执行入口，不是零散资料推荐页。

## 2. 三个岗位如何分工

| 岗位 | 主要责任 | 核心输入 | 核心输出 | 主要交接对象 |
| --- | --- | --- | --- | --- |
| 项目负责人 | 前后端、技术方案、进度、教师侧、站点收口、答辩口径 | 结构化知识资产、联调结果、站点代码与演示要求 | 首页入口、平台站点、教师侧展示、最终发布版本 | OCR 负责人、工作流负责人 |
| OCR 与资料电子化负责人 | 教材、试卷、讲义电子化，拆卡，打标签，入库验收 | 纸质资料、章节树、命名规范、元数据规范 | 素材台账、OCR 校对稿、结构化知识资产、批次记录 | 项目负责人、工作流负责人 |
| 工作流与联调负责人 | ADP 工作流、多 Agent 路由、变量透传、检索绑定、回归样例 | 提示词规范、知识资产标签、平台接口字段 | 联调配置、测试样例、通过标准、发布前回归结果 | 项目负责人 |

## 3. 一轮开发怎么串起来

### 3.1 起点：项目负责人先定口径

项目负责人先做 3 件事：

1. 确认本轮要演示的学生主闭环、教师侧增强和工作流能力。
2. 把首页入口、文档结构和交付话术先定下来。
3. 把知识库和工作流需要遵守的字段、标签和验收标准发给另外两位成员。

### 3.2 中段：OCR 负责人先把资料做成可检索资产

OCR 负责人不是只负责扫描，而是负责把原始资料变成后续能检索、能联调、能抽样验收的知识资产，包括：

1. 建素材台账。
2. 做扫描与 OCR。
3. 完成公式、题号、图表人工校对。
4. 按知识点、题目、课时、错因拆成结构化卡片。
5. 补全元数据和标签。
6. 形成可以交付给工作流与前端的入库批次。

### 3.3 中段：工作流负责人把知识资产接到 Agent 路由里

工作流负责人接手之后重点做：

1. 用统一状态槽位识别学生当前状态。
2. 把检索条件和知识资产标签绑起来。
3. 将 `TeacherOrchestrator / DiagnosisAgent / ExplanationAgent / PracticeEvalAgent / ReviewPlanAgent / TeacherOpsAgent` 的边界写清。
4. 输出联调样例、失败样例和通过标准。

### 3.4 收口：项目负责人把前端、阅读页和发布链路收回到一张图上

最后项目负责人负责把：

- 首页三个入口
- 文档检索和阅读页
- 高等数学示例知识库
- 平台文档
- GitHub Pages 发布

统一成一个能公开访问的版本。

## 4. 三个人之间最关键的交接链路

### 4.1 项目负责人 -> OCR 负责人

- 给课程范围、章节树、文件命名规则、最低元数据字段和批次验收标准。
- 明确哪些资料优先做电子化，哪些先作为附录。

### 4.2 OCR 负责人 -> 工作流负责人

- 交结构化知识资产。
- 交标签表和元数据样例。
- 交批次说明和抽样问答记录。

### 4.3 工作流负责人 -> 项目负责人

- 交 ADP 工作流配置说明。
- 交联调通过样例与失败样例。
- 交需要在前端和答辩中展示的关键流程。

### 4.4 项目负责人 -> 全体

- 统一最后一版首页入口、阅读路径和公开链接。
- 统一演示顺序、教师侧口径和答辩词。
- 统一中文提交信息和发布步骤。

## 5. 总体验收怎么做

一期收口时，三个人需要对同一份清单负责：

1. 首页只有 3 个主入口，并且都能打开正确文档。
2. 团队协作分类页能进入 3 份岗位文档。
3. 知识库主文档能进入高等数学示例附录。
4. 工作流文档能说清 Agent 边界、检索绑定和回归标准。
5. `npm run build` 通过。
6. `master` 分支内容与对外站点一致。
7. `npm run deploy:pages` 后 GitHub Pages 返回 `200`。
