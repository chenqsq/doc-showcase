# AI主导学习平台-平台需求与验收

> 文档层级：平台层  
> 文档目的：给出平台级功能需求、页面需求与验收口径  
> 核心结论：平台验收重点是学生能否被带入实时演化学习地图，并在学习、复习和资料进化中形成闭环

## 1. 平台级功能需求

| 编号 | 能力 | 说明 | 输出 |
| --- | --- | --- | --- |
| `PFR-01` | 选科与开学 | 支持单科或多科选择并创建学习启动会话 | `科目选择记录`、`学习启动会话` |
| `PFR-02` | 初始地图生成 | 基于知识资产和画像生成第一版学习地图 | `AI学习地图` |
| `PFR-03` | 短诊断校准 | 用轻量诊断校准起点和主线顺序 | `重规划事件` |
| `PFR-04` | 实时地图重规划 | 学习中动态插入补桥、复习、挑战和奖励节点 | `地图节点`、`重规划事件` |
| `PFR-05` | 闯关学习会话 | 支持讲解、作答、评分和下一步推进 | `闯关学习任务`、`成长反馈事件` |
| `PFR-06` | 学习画像更新 | 持续更新掌握度、薄弱基础和错误模式 | `学习画像` |
| `PFR-07` | 笔记与复习资产 | 自动生成思维导图、结构化笔记和复习计划 | `复习笔记包` |
| `PFR-08` | 资料注入与自动入库 | 支持资料识别、结构化、入库和演化记录 | `知识资产包`、`知识演化记录` |
| `PFR-09` | 自治分析后台 | 展示 Agent 状态、策略快照、日志和审计 | `策略快照` |

## 2. 页面验收

| 页面 | 验收要求 |
| --- | --- |
| 选科与开学页 | 能快速选科、开学并进入主流程 |
| AI学习地图页 | 能显示主线、支线、Boss、推荐下一步和重规划提示 |
| AI闯关学习页 | 能承接讲解、作答、评分、正反馈和下一步推进 |
| 笔记复习与成长页 | 能显示思维导图、结构化笔记、画像和成长曲线 |
| 资料注入与知识库演化后台 | 能显示资料任务、资产包和演化记录 |
| 系统自治与策略分析后台 | 能显示 Agent 状态、策略变化和异常审计 |

## 3. 业务验收

| 编号 | 验收条件 |
| --- | --- |
| `PAC-BIZ-01` | 学生点击“开始学习”后能进入 AI 学习地图 |
| `PAC-BIZ-02` | 短诊断后地图会重排 |
| `PAC-BIZ-03` | 学习中暴露基础缺口时能自动开补桥支线 |
| `PAC-BIZ-04` | 通关后能看到能力成长和地图推进 |
| `PAC-BIZ-05` | 学习中和学习结束后画像都能更新 |
| `PAC-BIZ-06` | 学习后能看到思维导图和结构化笔记 |
| `PAC-BIZ-07` | 新资料入库后能看到演化记录并影响地图 |
