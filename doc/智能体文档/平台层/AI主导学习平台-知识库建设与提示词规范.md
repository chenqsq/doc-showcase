# AI主导学习平台-知识库建设与提示词规范

## 与其他文档的边界

本文只负责回答：

1. 通用知识库应该怎样建设
2. 没有电子版的教材和试卷怎么转成知识资产
3. 提示词应该共享哪些状态槽位和回答模板

本文不替代具体学科附录，也不替代 ADP 节点配置说明。

## 一句话先记住

> 先把资料做成结构化知识资产，再把学生状态和回答模板统一下来，Agent 才能稳定地“检索对、讲得清、推进稳”。

## 1. 通用知识库建设流程

知识库建设固定按下面 8 步执行：

1. 原始素材登记
2. 扫描或拍照
3. OCR 初稿
4. 公式、题号、图表人工校对
5. 结构化拆卡
6. 元数据与标签标注
7. 批次入库
8. 抽样问答验收

这 8 步缺任意一步，资料都不应视为正式入库完成。

## 2. 非电子版资料怎么转

### 2.1 先做素材台账

素材台账至少包含：

- 来源
- 资料类型
- 学科
- 章节范围
- 原始形态
- 当前状态
- 负责人
- 批次编号

### 2.2 再做扫描与 OCR

统一要求：

- 文字页清晰，不裁边，不缺页。
- 公式、图像、题号密集页单独复检。
- OCR 初稿只作为中间稿，不能直接丢进知识库。

### 2.3 最后做人工校对

人工校对优先级：

1. 公式
2. 题号
3. 图表
4. 条件与结论
5. 答案与解析对应关系

## 3. 结构化拆卡规范

### 3.1 最小颗粒度

| 原始资料 | 不允许的做法 | 推荐拆分方式 |
| --- | --- | --- |
| 教材 | 整章直接入库 | 按知识点卡、定义卡、定理卡、例题卡拆分 |
| 试卷 | 整套卷子直接入库 | 按题目卡、题组卡、标准答案卡拆分 |
| 讲义 / PPT | 课件整份直接入库 | 按课时主题拆课堂重构笔记 |
| 错题 | 只放题面 | 拆成错题卡、误区卡、复盘卡 |

### 3.2 三类检索视角

知识资产必须至少服务 3 类检索视角：

- 学生问答：优先召回知识点卡、例题卡、错题卡
- 教师运营：优先召回教师摘要、错因总结、课堂重构
- 课堂重构：优先召回讲义、PPT、OCR 课时稿

## 4. 最低元数据字段

每份正式知识资产都必须补齐下面字段：

| 字段 | 作用 |
| --- | --- |
| `subject_category` | 统一学科大类 |
| `course_id` | 标识具体课程 |
| `module_id` | 标识模块 |
| `module_label` | 模块名称 |
| `chapter_id` | 标识章节 |
| `chapter_label` | 章节名称 |
| `resource_type` | 标识知识点卡、例题卡、课堂重构等类型 |
| `role` | 区分学生、教师或通用 |
| `source_type` | 标识教材、试卷、讲义、错题等来源 |
| `source_ref` | 能回溯到页码、卷号或原始出处 |
| `keywords` | 支持关键词召回 |
| `aliases` | 支持学生常见说法和别名 |
| `difficulty` | 标识难度层级 |
| `status` | 标识草稿、可用、已验收等状态 |

## 5. 提示词统一状态槽位

所有学科 Agent 至少共享下面 7 个状态槽位：

| 字段 | 含义 | 常见取值 |
| --- | --- | --- |
| `foundation_level` | 学生基础水平 | `zero_base / weak / normal / strong` |
| `current_goal` | 当前目标 | `concept_bridge / explain / practice / review / teacher_ops` |
| `current_chapter` | 当前模块或章节 | 如 `M02 / 导数定义` |
| `error_pattern` | 当前错因 | 如“只会套公式”“把极限当代入” |
| `confidence_level` | 当前把握度 | `low / medium / high` |
| `desired_depth` | 期望讲解深度 | `brief / standard / deep` |
| `turn_budget` | 本轮推进预算 | `1-step / 2-step / 3-step` |

## 6. 固定回答模板

### 6.1 `0基础补桥`

回答顺序固定为：

1. 先判断学生卡点
2. 用人话解释概念
3. 给最小例子
4. 只推进一步
5. 给单一下一步动作

### 6.2 `常规提分`

重点是：

- 快速识别当前知识点
- 给标准解题思路
- 明确这一题训练的是哪类方法
- 给一个可执行的小练习或下一步

### 6.3 `刷题纠错`

重点是：

- 先指出错因
- 再说明为什么错
- 然后给纠正步骤
- 最后给一题同类检查题

### 6.4 `教师运营摘要`

重点是：

- 汇总学生当前停滞点
- 汇总高频错因
- 提出补讲动作
- 明确数据边界，不能假装有真实班级画像

## 7. 知识库和提示词为什么要一起设计

因为下面这几件事是连在一起的：

- 检索标签决定 Agent 能拿到什么资料
- 状态槽位决定 Agent 该怎么说
- 回答模板决定 Agent 怎样把检索结果组织成教学话语

如果只做知识库，不统一提示词，回答会飘。

如果只做提示词，不做拆卡和标签，检索会飘。

## 8. 高等数学如何映射到这套规范

高等数学当前作为第一门完整示例学科，承担两件事：

1. 演示纸质教材、试卷、讲义怎样走完电子化到入库的完整链路。
2. 演示 `0基础补桥 / 常规提分 / 刷题纠错 / 教师运营摘要` 四类模板怎样在真实学科里落地。

## 9. 最终验收清单

知识库与提示词规范落地时，至少要确认：

1. 原始资料已经形成台账。
2. OCR 后的公式、题号和图表经过人工校对。
3. 正式知识资产已经拆到可检索颗粒度。
4. 元数据字段完整。
5. 三类检索视角可用。
6. 四类回答模板都能对应到明确场景。
7. 示例学科能用这套规范稳定跑通。
