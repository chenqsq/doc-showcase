# 比赛交付与答辩手册

> 文档层级：交付层  
> 文档目的：说明比赛版文档包怎么收口、怎么彩排、怎么给评委访问  
> 核心结论：交付层负责把学生主线演示、后台补充演示、访问方式和责任分工收成一套稳定执行手册

## 1. 当前交付包结构

比赛交付包统一收口到 `doc/作品文档`：

1. `00-作品总览与阅读地图`
2. `01-PRD与需求分析`
3. `02-场景与用户流程`
4. `03-页面与交互设计`
5. `04-总体架构与技术选型`
6. `05-算法与知识库设计`
7. `06-接口与API说明`
8. `07-测试验证与预期效果`
9. `08-答辩PPT大纲`
10. `09-演示视频脚本`
11. `10-访问与评测手册`
12. `11-开发技术文档`

## 2. 现场演示顺序

推荐固定顺序：

1. 选科与开学页
2. AI学习地图页
3. AI闯关学习页
4. 笔记复习与成长页
5. 资料注入与知识库演化后台
6. 系统自治与策略分析后台

## 3. 团队分工建议

| 角色 | 主要职责 |
| --- | --- |
| 项目负责人 | 控制术语一致性、文档收口、答辩口径 |
| 前端负责人 | 页面实现、地图交互、演示流畅性 |
| 后端负责人 | API、SSE、存储、权限与部署 |
| 算法负责人 | 地图生成、重规划、画像、笔记与入库策略 |
| 材料负责人 | PPT、视频脚本、访问手册与比赛格式收尾 |

## 4. 彩排检查表

- 作品名是否统一为 `AI主导学习生命周期的自进化自学智能体平台`
- 6 个页面是否都能打开
- 学生主线能否在 3 分钟和 5 到 10 分钟两种版本间切换
- 页面、PPT、视频脚本、访问手册中的术语是否一致
- 是否还残留旧的“多角色课堂工作台”口径
