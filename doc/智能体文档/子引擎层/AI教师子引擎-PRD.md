# AI教师子引擎-PRD

> 文档层级：子引擎层  
> 文档目的：定义 AI教师子引擎在比赛版系统中的职责、能力边界和验收重点  
> 核心结论：AI教师子引擎不只是答题器，它要同时承担知识重构、伴学执行、错题归因和教师洞察生成  
> 目标读者：后端开发、算法协作者、前端联调、产品负责人  
> 上游真源：[AI主导学习平台-总体架构设计.md](../平台层/AI主导学习平台-总体架构设计.md)、[AI主导学习平台-统一对象与接口契约.md](../平台层/AI主导学习平台-统一对象与接口契约.md)  
> 下游引用：[AI教师子引擎-技术方案.md](./AI教师子引擎-技术方案.md)、[05-算法与知识库设计.md](../../作品文档/05-算法与知识库设计.md)

## 与其他文档的边界

本文定义子引擎做什么、输出什么、怎么验收。  
平台编排、页面展示、数据库对象字段和比赛包装不在本文首定义。

## 一句话先记住

> 子引擎的职责是把“知识重构后的教学资产”真正变成学生可学、教师可看、系统可继续编排的结构化结果。

## 1. 子引擎正式目标

AI教师子引擎当前承担 4 类正式目标：

1. 解析课堂素材并完成知识重构
2. 围绕任务卡完成学生伴学会话
3. 生成掌握度快照与错题画像
4. 聚合教师学情洞察摘要

## 2. 双主线升级为四段链路

在原有“学生教学执行线 + 教师运营支持线”基础上，比赛版补全为四段链路：

1. 课堂知识重构段
2. 学生教学执行段
3. 错题复练与复习段
4. 教师洞察聚合段

## 3. 核心能力

| 能力 | 作用 | 关键输出 |
| --- | --- | --- |
| 多模态课堂解析 | 理解课件、讲义、题单、图片与文本 | 课堂结构化素材 |
| 知识点重构 | 生成知识点、先修关系、易错点和例题 | 知识重构结果 |
| 路径规划 | 结合先修图谱与任务目标生成学习路径 | 任务建议 |
| 流式伴学 | 在会话中完成讲解、追问、提示和作答反馈 | SSE 会话流 |
| 掌握度评估 | 生成知识点掌握评分和风险等级 | 掌握度快照 |
| 错题归因 | 识别错因类型并给出补桥方向 | 错题画像 |
| 变式生成 | 为同类错误生成复练题和间隔复习任务 | 复练任务 |
| 教师洞察聚合 | 汇总群体风险、热区知识点和干预建议 | 教师洞察摘要 |

## 4. 输入与输出

### 4.1 输入

- 课堂素材包
- 知识重构结果
- 学习会话
- 任务卡
- 学生作答与历史摘要
- `visitor_biz_id`
- `custom_variables`
- `chapter_id`
- `role`

### 4.2 输出

- 知识重构结果
- 会话流输出
- 掌握度快照
- 错题画像
- 教师洞察摘要

## 5. 功能需求（FR）

| 编号 | 能力 | 阶段 |
| --- | --- | --- |
| `FR-01` | 支持课堂资料多模态解析 | `P0` |
| `FR-02` | 支持知识点与先修图谱重构 | `P0` |
| `FR-03` | 支持基于任务卡的流式伴学对话 | `P0` |
| `FR-04` | 支持作答评分与追问纠偏 | `P0` |
| `FR-05` | 支持掌握度快照生成 | `P0` |
| `FR-06` | 支持错题归因和变式生成 | `P1` |
| `FR-07` | 支持间隔复习任务建议 | `P1` |
| `FR-08` | 支持班级级别的教师洞察聚合 | `P1` |
| `FR-09` | 支持面向前端工作台的 SSE 输出承接 | `P2` |
| `FR-10` | 支持平台单体服务通过 REST + SSE 接入 | `P2` |

## 6. 验收重点

子引擎通过标准至少包括：

- 能从课堂资料里抽出结构化知识结果，而不只是原文摘要
- 能围绕任务卡持续讲解和纠偏，而不是无边界聊天
- 能给出掌握度和错因，而不只是“答对/答错”
- 能给教师端输出可执行洞察，而不只是原始日志

## 7. 不做什么

- 不取代平台层做会话和权限主控
- 不单独定义前端页面结构
- 不把模型提示词当成正式产品契约
- 不把高数写成唯一学科边界

## 读完后你应该带走什么

- 子引擎已经从“教学执行器”升级成“知识重构 + 伴学 + 复练 + 教师洞察”的统一 AI 核心。
- 子引擎所有输入输出都必须回到平台统一对象。
- 比赛版页面和接口都已经有明确的子引擎承接边界。
