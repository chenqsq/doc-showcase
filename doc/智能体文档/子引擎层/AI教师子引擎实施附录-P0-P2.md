# AI教师子引擎实施附录-P0-P2

> 文档层级：子引擎层  
> 文档目的：把 `P0 / P1 / P2` 三条并行工作线压成一份实施附录，避免分散成三篇重复背景文档  
> 核心结论：`P0 / P1 / P2` 现在是并行工作线，不是串行章节；它们共同服务同一次公开交付  
> 目标读者：实现者、项目负责人、联调负责人  
> 推荐下一步：总览先看 [AI教师子引擎总览与设计.md](./AI教师子引擎总览与设计.md)

## 1. 为什么现在合成一篇

过去三篇附录都在重复讲：

- 这是同一期工作
- 它们之间有依赖
- 最终要一起收口

所以现在只保留一篇总附录，用来说明三条工作线各自负责什么。

## 2. P0：学生主闭环

### 目标

让学生能沿一轮完整链路完成：

- 诊断
- 讲解
- 练习
- 复盘

### 这条线最重要的产出

- 稳定的学生主线回答
- 能回流的结构化结果
- 一组最小闭环样例

## 3. P1：教师侧与展示增强

### 目标

在不破坏学生主闭环的前提下，让教师能看到：

- 学生风险
- 错因摘要
- 补讲建议
- 展示可用结果

### 这条线最重要的产出

- 教师运营摘要
- 课堂重构资产回流
- 对评委可展示的增强结果

## 4. P2：接入、联调与发布收口

### 目标

让平台、子引擎、知识资产和站点真正接起来，并可以公开发布。

### 这条线最重要的产出

- 联调通过记录
- 产品接入链路
- 发布前检查清单
- GitHub Pages 可访问版本

## 5. 当前优先级怎么排

1. 先稳住 `P0`。
2. 再保证 `P1` 至少有一条能展示的教师主线。
3. 最后由 `P2` 负责把两者收口到产品接入和发布。

这不是说 `P2` 最后才做，而是说 `P2` 的价值在于把前两条线收住。
