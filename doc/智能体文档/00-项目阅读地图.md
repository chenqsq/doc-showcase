## 3. 当前站点怎么分

| 区域 | 这一组主要放什么 | 什么时候看 |
| --- | --- | --- |
| 先读项目 | 阅读地图、文档总索引 | 刚进入项目时 |
| 平台核心 | 平台导读、产品总纲、总体架构设计、平台需求与验收 | 想先抓平台全貌时 |
| 平台规范 | 对象契约、知识库契约、生命周期、扩科规范 | 真正进入开发时 |
| 子引擎开发 | 导读、PRD、技术方案、教学策略、联调手册、P0-P2 附录 | 做 AI 教学工作流时 |
| 学科示例 | 导读、高数接入示范、落库规范、提示词、ADP、模板 | 需要看一门学科怎么落地时 |
| 团队与交付 | 团队分工、岗位手册、比赛对齐、答辩脚本、参考材料 | 需要收口、彩排和发布时 |
| 归档 | 历史版本 | 对照旧方案时 |

## 4. 高数知识库怎么读

不要一上来就扎进 100 多篇知识卡。

推荐顺序是：

1. 先看 [高等数学_测试-课程总览-高数_测试全景地图.md](../../kb/高等数学_测试/00-课程总览/高等数学_测试-00课程总览-CH00整门课程-课程总览-高数_测试全景地图.md)
2. 再看 [高等数学_测试-学习路径说明.md](../../kb/高等数学_测试/00-课程总览/高等数学_测试-00课程总览-CH00整门课程-章节导学-学习路径说明.md)
3. 再看 `R-课堂重构` 和 `T-教师运营`
4. 最后按 `M00` 到 `M10` 进入具体知识资产

也就是说，知识库还在，但它现在回到“知识资产层”，不再替项目主线做叙事。

## 5. 这轮恢复了什么

这轮恢复回主路径的，不是边角料，而是开发真正会用到的文档，包括：

- 平台层的产品总纲、总体架构设计、平台需求与验收
- 平台层的统一对象与接口契约、知识库结构与契约、知识库建设与提示词规范
- 子引擎层的 PRD、技术方案、教学策略设计、联调与验收手册、P0-P2 实施附录
- 学科层的接入示范、落库规范、提示词规范、ADP 配置和学科接入模板
- 团队与交付层的团队分工、岗位手册、比赛对齐说明、答辩口径与演示脚本

所以现在的站点不是“几篇合并稿”，而是“导读层 + 开发层”一起工作。
