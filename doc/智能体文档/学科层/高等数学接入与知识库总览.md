# 高等数学接入与知识库总览

> 文档层级：学科导读  
> 文档目的：告诉你高数示例这一组文档怎么读，以及知识库导航层和资产层怎么区分  
> 核心结论：这篇只负责找路；高数平台接入、落库规范、提示词规范、ADP 配置和知识库导航文档都继续保留为独立真源  
> 目标读者：学科接入实施者、知识库负责人、工作流联调负责人、评委  
> 推荐下一步：先读 [高等数学-平台接入示范.md](./高等数学-平台接入示范.md)，要做知识库就继续看 [高等数学-知识库接入与落库方案.md](./高等数学-知识库接入与落库方案.md)

## 1. 这组文档解决什么

高数示例这一组主要解决 4 件事：

1. 为什么先用高等数学做第一门示范学科
2. 高数怎样接进平台和子引擎
3. 高数知识库怎样从资料整理、拆卡、标签到落库
4. 提示词和 ADP 配置怎样落到具体学科

## 3. 高数知识库本身怎么读

知识库还在，而且没有删。

推荐顺序是：

1. [高等数学_测试-课程总览-高数_测试全景地图.md](../../kb/高等数学_测试/00-课程总览/高等数学_测试-00课程总览-CH00整门课程-课程总览-高数_测试全景地图.md)
2. [高等数学_测试-学习路径说明.md](../../kb/高等数学_测试/00-课程总览/高等数学_测试-00课程总览-CH00整门课程-章节导学-学习路径说明.md)
3. `R-课堂重构`
4. `T-教师运营`
5. `M00` 到 `M10` 的具体知识资产

也就是说：

- 课程总览和学习路径说明属于导航层
- 课堂重构、教师运营和模块知识卡属于资产层
- 资产层默认不承担项目主叙事，但仍然是演示和检索的真实内容来源
