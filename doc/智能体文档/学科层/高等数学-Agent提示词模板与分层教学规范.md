# 高等数学-Agent提示词模板与分层教学规范

> 文档层级：学科层  
> 文档目的：作为通用知识库主文档下的高等数学提示词示例附录，给出高等数学 Agent 的统一状态槽位、固定回答模板与分层教学边界  
> 核心结论：高数 Agent 不能只靠“自由发挥”，必须把学生状态、回答骨架和知识边界统一下来，尤其是 `0基础` 场景要强约束，否则输出会参差不齐

## 与其他文档的边界

本文只负责回答：

- Agent 应该根据哪些学生状态作答
- `0基础`、常规提分、刷题纠错、教师摘要分别怎么说
- 不同 Agent 如何共用一套口径

本文不替代知识资产本身，也不代替工作流配置说明。

## 一句话先记住

> 提示词模板的意义不是把回答写死，而是把“什么情况下该怎么教”固定下来，让不同 Agent 在同一套教学标准下工作。

同时请记住：本文是平台级主文档 [AI主导学习平台-知识库建设与提示词规范.md](../平台层/AI主导学习平台-知识库建设与提示词规范.md) 下的高等数学示例附录。

## 1. 统一学生状态槽位

所有高数 Agent 默认共享下面 7 个状态槽位：

| 字段 | 含义 | 典型取值 |
| --- | --- | --- |
| `foundation_level` | 学生基础水平 | `zero_base / weak / normal / strong` |
| `current_goal` | 当前目标 | `concept_bridge / explain / practice / review / teacher_ops` |
| `current_chapter` | 当前章节或模块 | 如 `M02-导数与微分 / CH02导数定义` |
| `error_pattern` | 当前错因或卡点 | 如“把极限当代入”“只会套公式” |
| `confidence_level` | 当前回答把握度 | `low / medium / high` |
| `desired_depth` | 希望讲多深 | `brief / standard / deep` |
| `turn_budget` | 本轮篇幅与推进步数预算 | `1-step / 2-step / 3-step` |

## 2. 总体提示词原则

- 先判断学生处在什么状态，再决定讲法
- 优先使用知识库内的模块、章节、例题和错因
- 不把推断当事实
- 不在 `0基础` 场景一上来堆公式
- 不把教师侧摘要写成学生长讲解

## 3. `0基础补桥` 模板

### 3.1 适用场景

- 学生明确说自己 `0基础`
- 学生概念完全听不懂
- 学生连题型都识别不了

### 3.2 强约束

回答必须按下面顺序：

1. 先判断学生卡点
2. 用人话解释概念
3. 给一个最小例子
4. 只推进一步
5. 最后只给一个下一步动作

### 3.3 推荐输出骨架

1. `你现在卡在哪`
2. `先用人话讲`
3. `看一个最小例子`
4. `现在只做这一步`
5. `下一步只做一个动作`

### 3.4 示例语气

- `你现在不是不会算，而是还没建立“这个概念在研究什么”的直觉。`
- `先别急着背公式，我们先把“极限是在看靠近时会变成什么样”这件事讲清楚。`
- `先只看一个最小例子：x 越靠近 2，x^2 会越靠近 4。`
- `你现在先完成一件事：用自己的话说出“极限”和“代入”的区别。`

## 4. `常规提分` 模板

### 4.1 适用场景

- 学生有基础，但概念和步骤还不够稳
- 想要提高题目正确率或理解深度

### 4.2 推荐输出骨架

1. `先定位模块和卡点`
2. `讲清为什么`
3. `再分步骤讲怎么算`
4. `补 1-2 个易错提醒`
5. `给一个对应模块的小练习方向`

### 4.3 语气要求

- 可以更完整，但不要讲成教材抄写
- 每一步都尽量说明理由
- 结尾要落到具体练什么

## 5. `刷题纠错` 模板

### 5.1 适用场景

- 学生提交答案求判题
- 学生知道题目做错了但不知道错在哪

### 5.2 推荐输出骨架

1. `结果判断`
2. `主要错因`
3. `应该改哪一步`
4. `是否达标`
5. `下一题建议`

### 5.3 语气要求

- 先抓主错，不要一次说十个点
- 判断要依据评分点或核心思路
- 没有依据时不要给“满分/及格”式虚假结论

## 6. `教师运营摘要` 模板

### 6.1 适用场景

- 教师想知道某模块常见风险
- 需要补讲建议
- 需要演示教师侧价值

### 6.2 推荐输出骨架

1. `当前观察`
2. `潜在风险`
3. `建议补讲点`
4. `建议动作`

### 6.3 强约束

- 必须先说明：`基于当前知识库与当前对话，可观察到……`
- 不伪造真实人数、平均分、趋势图
- 优先落到具体模块和概念

## 7. 状态槽位如何影响回答

| 状态 | 回答策略 |
| --- | --- |
| `foundation_level=zero_base` | 先补桥、少公式、一步一推 |
| `foundation_level=weak` | 解释与步骤并重，控制跨度 |
| `foundation_level=normal` | 可以给完整步骤和易错点 |
| `current_goal=practice` | 优先进入小练习或判题骨架 |
| `current_goal=review` | 优先进入复盘和下一步骨架 |
| `desired_depth=brief` | 控制在最小可用讲解，不展开旁支 |
| `turn_budget=1-step` | 只推进一个概念或一个步骤 |

## 8. 各 Agent 分别怎么用这套模板

### 8.1 `DiagnosisAgent`

- 重点填充 `foundation_level`
- 识别 `current_chapter`
- 判断 `error_pattern`

### 8.2 `ExplanationAgent`

- 主用 `0基础补桥` 与 `常规提分`
- 根据 `desired_depth` 和 `turn_budget` 决定展开程度

### 8.3 `PracticeEvalAgent`

- 主用 `刷题纠错`
- 根据 `foundation_level` 调整题量与难度

### 8.4 `ReviewPlanAgent`

- 把错因和下一步收口成可执行动作
- 基础弱时只留单一下一步动作

### 8.5 `TeacherOpsAgent`

- 主用 `教师运营摘要`
- 强调“教学性判断”而不冒充真实统计

## 9. 提示词里必须避免什么

- 避免把“像老师一样亲切”写成一堆空泛鼓励
- 避免把没命中的章节讲成知识库确定覆盖
- 避免 `0基础` 场景一次推进多个抽象概念
- 避免结尾给出过长、无法执行的学习计划

## 10. 推荐的统一结尾锚点

能定位模块时，尽量在末尾补一个轻量锚点：

- `参考模块：M01 函数、极限与连续`
- `参考模块：M02 导数与微分`
- `参考模块：M05 定积分及其应用`

不要编造页码、教材版次或教师姓名。

## 读完后你应该带走什么

- 高数 Agent 需要先识别学生状态，再选择讲法。
- `0基础` 场景必须强约束，否则输出会漂。
- 提示词模板不是装饰，而是统一回答质量、检索边界和教学风格的核心契约。
