# AI主导学习生命周期的自进化自学智能体平台：作品总览与阅读地图

> 文档层级：作品主文档  
> 文档目的：统一作品定义、阅读顺序、交付边界和比赛材料映射  
> 核心结论：这不是“会答题的聊天页”，而是一套由 AI 接管学生学习生命周期、能够实时演化学习地图并持续自我进化的自学智能体平台  
> 技术真源：[AI主导学习平台-产品总纲.md](../智能体文档/平台层/AI主导学习平台-产品总纲.md)、[AI主导学习平台-总体架构设计.md](../智能体文档/平台层/AI主导学习平台-总体架构设计.md)、[AI教师智能体群引擎-PRD.md](../智能体文档/子引擎层/AI教师智能体群引擎-PRD.md)

## 一句话先记住

> 学生只需要选科并点击“开始学习”，平台就会自动生成学习地图、发起短诊断、实时调整关卡、更新画像、生成笔记，并在新资料进入后让知识库和教学策略一起进化。

## 1. 作品定义

作品唯一正式名称：

`AI主导学习生命周期的自进化自学智能体平台`

这套系统固定证明 4 件事：

1. AI 能接管学习生命周期，而不是只在学生提问时被动响应。
2. 学习目录不是静态课程表，而是会根据学生状态持续演化的学习地图。
3. 学习过程中会持续沉淀画像、笔记、思维导图和复习资产。
4. 学生或平台管理者上传的新资料，会被 AI 自动识别、结构化、入库，并反向影响后续学习地图。

## 2. 作品现在在讲什么

当前比赛口径固定围绕这条主链展开：

`选科 -> 开始学习 -> AI 初始学习地图 -> 短诊断校准 -> 实时闯关学习 -> 画像更新 -> 笔记与思维导图 -> 资料入库 -> 地图再演化`

其中最重要的展示点有 5 个：

- `AI 接管开学`
- `实时演化学习地图`
- `闯关式即时正反馈`
- `学习画像持续更新`
- `知识库与策略自进化`

## 3. 六个核心页面一眼看懂

| 页面 | 一句话用途 |
| --- | --- |
| 选科与开学页 | 选择一门或多门科目，点击“开始学习”，把学生送入 AI 接管流程 |
| AI学习地图页 | 看见当前主线、补桥支线、阶段 Boss、推荐下一步和实时重规划结果 |
| AI闯关学习页 | 真正上课和闯关的地方，完成讲解、作答、反馈和推进 |
| 笔记复习与成长页 | 集中查看思维导图、结构化笔记、错题回顾、复习计划和成长曲线 |
| 资料注入与知识库演化后台 | 查看资料识别、结构化、入库和对学习地图的影响 |
| 系统自治与策略分析后台 | 查看 Agent 协同、策略版本、画像更新日志、异常告警和回滚审计 |

## 6. 当前默认边界

| 项目 | 当前默认值 |
| --- | --- |
| 第一示范学科 | 高等数学 |
| 科目能力 | 支持多科并行计划 |
| 启动方式 | 先生成初始地图，再做短诊断校准 |
| 地图形态 | 实时演化学习地图，不允许写成人工静态目录 |
| 页面风格 | 中强度闯关化，不做重度游戏皮肤 |
| 部署形态 | 单机服务，不走微服务 |

## 7. 读完本文后你应该带走什么

- 作品名、页面名和真源口径已经统一。
- 学生主线才是比赛主叙事，后台只负责证明平台会自己长。
- “目录”现在是平台最核心的智能表现，不是附属 UI。
- 真正要证明的不是 AI 会讲题，而是它能持续组织学习、持续纠偏、持续进化。
