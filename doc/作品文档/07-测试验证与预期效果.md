# AI主导学习生命周期的自进化自学智能体平台测试验证与预期效果

> 文档层级：作品主文档  
> 文档目的：定义验证口径、覆盖矩阵和预期效果  
> 核心结论：验收重点不再是“教师页能不能看”，而是学生是否真的被带进一张会持续演化的学习地图里

## 1. 核心验收主线

标准链路固定为：

`选高数/多科 -> 开始学习 -> 初始地图 -> 短诊断 -> 地图重排 -> AI闯关 -> 作答 -> 正反馈 -> 画像更新 -> 生成思维导图/笔记 -> 新资料入库 -> 地图受影响`

## 6. 关键场景专项验证

### 6.1 实时重规划验证

- 学生连续错误时能立即插入补桥关卡
- 学生达标后能回到原主线
- 默认看到轻提示，展开后能看到完整原因

### 6.2 多科并行验证

- 同时选两门课时，能看到各自独立地图
- 能看到平台给出的今日学习队列
- 能看到跨科复习提醒

### 6.3 笔记区验证

- 单关结束后有摘要
- 一轮学习结束后有结构化笔记
- 阶段结束后有思维导图与复习计划

### 6.4 后台自治验证

- 新资料上传后有完整任务状态
- 入库后能看到演化记录
- 地图或节点受影响时能被追溯

## 7. 预期效果

### 对学生

- 开始学习的门槛明显下降
- 学习过程中不容易丢失方向
- 卡住时能被系统即时接住
- 学完后有真正可复习的笔记和思维导图

### 对平台管理者

- 新资料能快速进入知识库
- 地图和策略变化可追踪、可审计
- 不需要手动维护复杂的静态目录

### 对评委

- 能清楚看到“AI 接管学习生命周期”的系统性
- 能看到“地图实时演化”而不是只会对话
- 能看到“新资料进来后系统会变”这一竞争力
