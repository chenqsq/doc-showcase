# AI主导学习生命周期的自进化自学智能体平台演示视频脚本

> 文档层级：作品主文档  
> 文档目的：提供 3 分钟内的演示脚本  
> 核心结论：视频镜头以学生主线为主，后台放在后半段证明“平台会自己长”

## 1. 视频总时长

控制在 `180s` 内。

## 2. 镜头脚本

| 时间 | 画面 | 口播建议 |
| --- | --- | --- |
| `0s-12s` | 学生面对一门课不知道从哪学的碎片场景 | “自学最难的，不是找不到答案，而是不知道从哪开始，也不知道卡住时该怎么办。” |
| `12s-24s` | 打开平台首页，勾选高等数学或多科 | “这套平台让学生先选科，再由 AI 接管整个学习生命周期。” |
| `24s-42s` | 点击开始学习，出现 AI 初始学习地图 | “开始学习后，系统不会丢给学生一个空白聊天框，而是先生成一张可见的学习地图。” |
| `42s-60s` | 完成短诊断，地图重排 | “系统会用短诊断快速校准起点，让不同基础的学生进入不同的起始路径。” |
| `60s-95s` | 进入 AI 闯关学习页，流式讲解、作答 | “每个节点都像一个关卡，AI 会讲解、追问、判题，并持续推进当前任务。” |
| `95s-118s` | 故意答错，触发补桥支线和轻提示 | “当系统发现基础不足时，地图会实时开出补桥支线，并告诉学生为什么要绕这一段。” |
| `118s-136s` | 补桥通过后回主线，出现能力成长与新区域解锁 | “通过后，系统会把学生接回主线，并给出即时的成长反馈和新区域解锁。” |
| `136s-154s` | 打开笔记复习与成长页，展示思维导图和结构化笔记 | “学习结束后，Agent 群会自动生成思维导图、结构化笔记和复习计划。” |
| `154s-170s` | 打开资料注入后台，上传新资料，查看知识演化记录 | “学生或管理者上传的新资料，会被自动识别、入库，并形成清晰的知识演化记录。” |
| `170s-180s` | 回到地图页，展示地图已受影响 | “这说明平台不是死的课程表，而是一套会随着资料和学生状态持续进化的学习系统。” |

## 3. 视频中必须出现的能力

1. 选科与开始学习
2. AI 初始学习地图
3. 短诊断校准
4. 学习中实时重规划
5. 即时正反馈
6. 画像更新
7. 思维导图与结构化笔记
8. 新资料入库后地图受影响

## 4. 拍摄建议

- 学习地图页要停留 3 到 5 秒，让评委看清主线和支线
- 补桥触发后，务必保留“AI 已为你调整路线”的提示镜头
- 即时正反馈要能看到地图推进和解锁，不只展示一句文字
- 笔记区至少展示 1 张思维导图和 1 组结构化笔记
- 后台镜头不要太长，重点是证明“新资料真的会改变系统”
