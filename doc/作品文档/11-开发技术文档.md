# AI主导学习生命周期的自进化自学智能体平台开发技术文档

> 文档层级：作品主文档  
> 文档目的：说明平台选择、关键实现、部署方式和二次开发边界  
> 核心结论：比赛版技术实现遵循“单体清晰、结构稳定、AI 友好维护、能力可持续演进”的原则

## 1. 技术栈总览

### 前端

- `Vue 3`
- `TypeScript`
- `Vite`
- `Vue Router`
- `Pinia`
- `Naive UI`
- `Tailwind CSS`
- `VueUse Motion`
- `ECharts`

### 后端

- `Go 1.24`
- `Gin`
- `pgx + sqlc`
- `PostgreSQL 16`
- `腾讯 COS / MinIO`
- `腾讯 ADP + HTTP SSE V2`

## 2. 核心模块

| 模块 | 负责什么 |
| --- | --- |
| 启动与选科模块 | 选科、多科优先级、学习启动会话 |
| 学习地图模块 | 生成、查询、节点状态、重规划事件 |
| 闯关学习模块 | 讲解、作答、评分、反馈、推进 |
| 学习画像模块 | 增量更新画像和成长曲线 |
| 笔记模块 | 生成思维导图、结构化笔记和复习计划 |
| 资料注入模块 | 识别、切分、标注、入库、演化记录 |
| 自治分析模块 | Agent 状态、策略快照、日志和审计 |

## 3. Agent 群实现建议

| Agent | 最小职责 |
| --- | --- |
| `StarterAgent` | 生成初始地图 |
| `DiagnosisAgent` | 执行短诊断 |
| `MapPlannerAgent` | 负责地图重规划 |
| `TutorAgent` | 负责讲解和引导 |
| `EvaluatorAgent` | 负责评分和成长反馈 |
| `ProfileAgent` | 负责画像更新 |
| `NoteSynthAgent` | 负责笔记和思维导图 |
| `IngestionAgent` | 负责资料识别和入库 |
| `StrategyAgent` | 负责策略快照和日志分析 |

## 4. 部署建议

### 单机结构

- 1 个 Vue 前端静态站点
- 1 个 Go 单体服务
- 1 个 PostgreSQL 数据库
- 1 个 COS / MinIO 文件存储

### 运行建议

- 使用 `Caddy / Nginx` 反向代理
- 使用 `systemd` 守护 Go 服务
- 保留数据库备份和恢复脚本
- 为流式接口和后台任务保留健康检查

## 5. 二次开发边界

### 允许扩展

- 新学科接入
- 多科排程策略增强
- 地图样式与动画升级
- 新的画像字段
- 新的笔记生成模板

### 当前不前置

- 微服务拆分
- Redis / MQ
- 自动微调底层模型
- 重游戏引擎式前端
