# 知脉课堂开发技术文档

> 文档层级：交付文档  
> 文档目的：作为比赛可选技术文档，收口平台选择、关键实现、部署方式与二次开发指引  
> 核心结论：知脉课堂的技术实现遵循“单体清晰、接口稳定、前后端职责明确、AI 友好维护”的原则

## 1. 平台选择说明

### 前端

- `Vue 3 + TypeScript + Vite`
- `Pinia`
- `Naive UI`
- `Tailwind CSS`
- `VueUse Motion`
- `ECharts`

选择原因：适合快速搭建真实工作台，组件完备，状态清晰，便于比赛期持续迭代。

### 后端

- `Go 1.24`
- `Gin`
- `pgx + sqlc`
- `PostgreSQL 16`

选择原因：单体结构简单、性能稳定、类型边界清晰，适合 AI 辅助持续开发。

### 智能体与接入

- 腾讯 `ADP`
- `HTTP SSE V2`
- `COS / MinIO`

## 2. 关键实现细节

| 模块 | 实现重点 |
| --- | --- |
| 素材接入 | 文件上传、格式校验、解析任务状态追踪 |
| 知识重构 | 课堂摘要、关系图、例题与复盘建议生成 |
| 伴学会话 | 流式讲解、上下文续接、作答反馈 |
| 错题复练 | 错因标签、变式题生成、复习调度 |
| 教师洞察 | 班级聚合、风险排序、补讲建议 |

## 3. 数据对象落地建议

建议至少建立以下表或集合：

- `courses`
- `chapters`
- `material_packs`
- `reconstruction_results`
- `learning_sessions`
- `mastery_snapshots`
- `wrong_question_profiles`
- `teacher_insights`
- `access_credentials`

## 4. 部署说明

### 推荐部署形态

- `Caddy/Nginx` 作为反向代理
- `Go` 服务作为单体 API 与 SSE 代理
- `PostgreSQL 16` 作为主数据库
- `COS` 作为正式文件存储

### 运维建议

- systemd 守护
- 每日数据库备份
- 对象存储版本化
- 健康检查与优雅重启

## 5. 二次开发指引

### 新增学科

1. 扩充课程与章节结构
2. 补充知识资产与错题样本
3. 配置新的学科提示词和检索标签
4. 复用现有页面与接口对象

### 扩展页面

1. 保持当前 6 页工作台结构
2. 复用任务卡、掌握度、错题画像等公共对象
3. 优先扩展模块，不重写全站导航

### 扩展接口

1. 继续沿用 `REST + SSE`
2. 新接口先落到公共对象，再映射平台字段
3. 严禁前端直接持有 `AppKey`
