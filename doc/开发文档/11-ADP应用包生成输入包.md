# ADP 应用包生成输入包

> 文档层级：工程参考层  
> 文档目的：把比赛作品主文档压缩成一套适合腾讯“应用包生成助手”消费的输入资产，减少生成结果跑偏成“聊天机器人”或“泛平台介绍”的概率  
> 当前口径：`完整平台版`、`高数优先兼容扩科`  
> 事实边界：本文件描述的是“输入包与提示词资产”，不是 ADP 控制台真实配置完成，也不代替真实导入、真实发布与真实联调证据

## 1. 为什么现在需要这份资产

`2026-04-12` 已经有“应用包生成助手 V2.1 第一版可用草案”的验收基线，但那一版更像“ADP 应用核心草案 + 平台集成合同草案”的起点。  
如果后续继续把整堆作品文档原文直接喂给腾讯助手，很容易出现 4 类偏差：

- 把作品生成成聊天机器人
- 把业务真状态塞进长期记忆
- 把新资料写成上传即生效
- 把多个 Agent 的职责重新糊成一个主控

所以这次新增一套“应用包生成输入包”，先把业务真源压缩成适合生成助手消费的格式，再做分轮生成。

## 4. 固定生成顺序

### 4.1 第一轮：应用包骨架生成

先用：

`10-主提示词-应用包骨架生成.md`

然后依次粘贴：

- `A-产品主线包.md`
- `B-Agent与工作流包.md`
- `C-接口契约包.md`
- `D-教学提示词风格包.md`

这一轮只要：

- 应用定位
- Agent 列表
- 学生主工作流
- 新资料工作流
- 变量清单
- 长期记忆边界
- 工具与知识库使用原则

不要直接让腾讯助手产出长篇 Agent 提示词。

### 4.2 第二轮：逐 Agent 生成提示词

先用：

`11-Agent批量定义提示词.md`

确认 Agent 拆分没跑偏之后，再用：

`12-单Agent细化提示词模板.md`

逐个生成：

- `StarterAgent`
- `DiagnosisAgent`
- `MapPlannerAgent`
- `TutorAgent`
- `EvaluatorAgent`
- `ProfileAgent`
- `NoteSynthAgent`
- `IngestionAgent`
- `StrategyAgent`

只有在腾讯模板硬性要求时，才补一个轻量 `TeacherOrchestrator`。

### 4.3 第三轮：平台集成合同生成

用：

`13-平台集成合同提示词.md`

重点让腾讯助手补齐：

- `VisitorId`
- `ConversationId`
- `SYS.Memory`
- `custom_variables`
- 工作流节点输入输出
- SSE 事件口径
- 知识治理状态流
- 审计与回滚约束

### 4.4 第四轮：回炉修正

如果腾讯助手的结果出现“像聊天机器人”“把状态塞进记忆”“上传即生效”“Agent 分工不清”等问题，用：

`14-修订提示词.md`

做定向修正，而不是整份推倒重来。

## 5. 生成时必须锁死的默认约束

- 应用模式：`Multi-Agent`
- 协同方式：`工作流编排`
- 学生主链：`选科 -> 初始地图 -> 短诊断 -> 当前关卡教学 -> 作答评分 -> 地图重规划 -> 画像更新 -> 笔记生成`
- 新资料链路：`上传 -> SaveDoc/OCR -> 声明抽取 -> candidate -> validation -> release/rollback -> 影响域重算`
- 长期记忆只存偏好、薄弱点、错误模式、节奏特征
- 长期记忆不存地图真状态、节点完成状态、知识发布状态
- 首个演示学科固定为 `高等数学`
- 变量与章节命名要保留扩科能力
- 输出语言固定中文
- 不生成营销首页，不假装完整上线成品

## 6. 最终验收清单

每次生成后都至少复核下面这些点：

- 6 个页面都能映射到对应 Agent 或工作流节点
- 学生主链完整覆盖地图、诊断、闯关、评分、反馈、画像、笔记
- 后台主链完整覆盖资料入库、候选区、校验、发布/回滚、策略快照、审计
- `LearningMap`、`RerouteEvent`、`KnowledgeRelease` 等真状态没有被塞进长期记忆
- 至少保留“极限/导数”这类高数示范口径
- 变量命名没有被锁死成只能高数
- 没有误生成营销首页、泛问答助手或自由转交主链
- 评分类 Agent 有结构化输出，不只返回自然语言

## 7. 和真实联调的边界

这套输入资产只是把文档主线压缩成适合腾讯助手消费的格式。  
真正开始做 ADP 控制台配置、知识库接入、SaveDoc 联调、发布检查时，仍然要遵守当前仓库已写明的协作前置：

- 你先登录 ADP 控制台并保持会话有效
- 我再接手浏览器做配置、核查、联调和取证
- 账号口令、密钥、Token 不写入仓库
