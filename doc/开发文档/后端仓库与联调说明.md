# 后端仓库与联调说明

`chenqsq/ai-learning-platform-backend` 是比赛版后端的独立仓库，用来承接 `AI主导学习生命周期的自进化自学智能体平台` 的 FastAPI 工程骨架。  
文档展示站仓库 `chenqsq/doc-showcase` 继续负责作品与工程文档，不直接承载后端代码。

| 后端仓库 | [chenqsq/ai-learning-platform-backend](https://github.com/chenqsq/ai-learning-platform-backend) |
| 文档站仓库 | [chenqsq/doc-showcase](https://github.com/chenqsq/doc-showcase) |

## 3. 当前骨架包含什么

- `Python 3.12 + FastAPI + SQLAlchemy + Alembic + psycopg + pydantic-settings + httpx + sse-starlette + pytest`
- 固定目录分层：`app/api`、`app/core`、`app/db`、`app/models`、`app/schemas`、`app/repositories`、`app/services`、`app/gateways`、`tests`、`alembic`
- 已注册 OpenAPI 主路径：
  - `/api/startup/subjects`
  - `/api/startup/resume`
  - `/api/startup/session`
  - `/api/diagnostics/{subjectId}/submit`
  - `/api/learning/tasks/{taskId}`
  - `/api/learning/sessions`
  - `/api/learning/sessions/{taskSessionId}/stream`
  - `/api/learning/sessions/{taskSessionId}/answers`
  - `/api/learning/sessions/{learningSessionId}/feedback`
  - `/api/ingestion/upload`
  - `/api/ingestion/tasks`
  - `/api/knowledge/candidates`
  - `/api/knowledge/releases`
  - `/api/knowledge/releases/{releaseId}`
  - `/api/ops/agents/status`
  - `/api/ops/strategies/latest`
  - `/api/ops/reroutes/logs`
  - `/api/ops/audit-events`
  - `/api/ops/healthz`
- 统一响应包络固定为 `success / traceId / data / message / error`
- 统一错误码前缀固定为 `STARTUP_ / MAP_ / LEARNING_ / PROFILE_ / NOTE_ / KNOWLEDGE_ / OPS_`
- 首个 Alembic 迁移已经落下比赛版最小主链基础表

## 4. 当前可运行范围

- `/api/ops/healthz` 已可用，用于确认 API、数据库、ADP gateway、对象存储 gateway 的基础状态
- 除健康检查外，其余业务路由当前统一返回标准化 `501 Not Implemented`
- 所有占位接口都保留了对应的 schema、service、repository、gateway 签名，后续可直接接真实实现

这意味着当前版本适合做三件事：

1. 统一工程结构和接口壳层。
2. 提前打通数据库迁移、应用启动和 OpenAPI。
3. 给前后端联调、ADP 接入和知识治理实现留稳定挂点。

## 5. 本地启动方式

1. 进入后端仓库目录并创建虚拟环境：

```powershell
cd E:\简历\ai-learning-platform-backend
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -e .[dev]
```

2. 复制环境变量模板：

```powershell
Copy-Item .env.example .env
```

3. 启动 PostgreSQL：

```powershell
docker compose up -d postgres
```

说明：为了避开本机常见的 `5432` 冲突，示例编排把宿主机端口映射为 `55432`，`.env.example` 也已经对齐这个端口。

4. 执行数据库迁移：

```powershell
alembic upgrade head
```

5. 启动 FastAPI：

```powershell
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

6. 打开接口文档和健康检查：

- Swagger UI: `http://localhost:8000/docs`
- Health Check: `http://localhost:8000/api/ops/healthz`

## 6. 环境变量说明

| 变量 | 用途 |
| --- | --- |
| `APP_ENV` | 运行环境标识，如 `local`、`dev`、`prod` |
| `APP_HOST` / `APP_PORT` | FastAPI 服务监听地址与端口 |
| `DATABASE_URL` | 数据库完整连接串，优先级高于拆分字段 |
| `POSTGRES_HOST` / `POSTGRES_PORT` / `POSTGRES_DB` / `POSTGRES_USER` / `POSTGRES_PASSWORD` | PostgreSQL 拆分配置 |
| `CORS_ORIGINS` | 允许跨域的前端来源列表 |
| `ADP_BASE_URL` / `ADP_APP_KEY` / `ADP_APP_ID` | 腾讯 ADP 接入配置 |
| `COS_BUCKET` / `COS_REGION` / `COS_SECRET_ID` / `COS_SECRET_KEY` / `COS_ENDPOINT` | 对象存储配置，兼容 COS / MinIO 场景 |
| `LOG_LEVEL` | 应用日志级别 |

如果 `ADP_*` 或 `COS_*` 没有填写，应用仍然可以启动，只是对应 gateway 会保持未启用状态，这也是当前骨架允许本地先跑起来的原因。
