# 提示词｜平台集成合同生成

请基于我已提供的 4 个输入包，以及已经确认的应用骨架和 Agent 拆分方案，生成一份 `平台集成合同草案`。

这份合同不是产品介绍文案，而是给平台接入、后端联调、知识治理和审计使用的结构化合同。

请按下面 7 个部分输出：

## 1. ADP 运行时标识与映射

请给出表格，明确：

- `VisitorId`
- `ConversationId`
- `SYS.Memory`
- `custom_variables`

分别对应什么、由谁生成、由谁保存、谁不能直接持有。

## 2. 学生主工作流节点合同

请按节点列出：

- 节点名
- 对应 Agent
- 输入对象
- 输出对象
- 是否需要结构化回写
- 是否需要 SSE

主链至少覆盖：

`选科 -> 初始地图 -> 短诊断 -> 当前关卡教学 -> 作答评分 -> 地图重规划 -> 画像更新 -> 笔记生成`

## 3. 资料入库工作流节点合同

请按节点列出：

- 上传
- SaveDoc/OCR
- 声明抽取
- 候选区
- 校验
- 发布
- 回滚
- 影响域重算

每个节点都要写清：

- 输入
- 输出
- 状态
- 审计要求

## 4. SSE 事件合同

请定义前端可消费的流式事件，至少包括：

- `tutor_delta`
- `checkpoint_prompt`
- `feedback_summary`
- `reroute_event`
- `done`

每个事件请给出：

- 事件名
- 触发节点
- 最小数据字段
- 前端用途

## 5. 知识治理状态机

请把下面这条链写成清晰合同：

`uploaded -> parsing -> extracted -> validating -> candidate -> released / archived / failed`

并明确：

- 何时允许进入 `candidate`
- 何时允许生成 `KnowledgeRelease`
- 何时必须回滚或归档

## 6. 审计、回滚与风险控制

请明确：

- 哪些动作必须写审计日志
- 哪些动作必须可回滚
- 哪些状态不能只存在于 ADP 记忆
- 哪些异常要在后台可见

## 7. 默认实现边界

请明确：

- 前端不能直连 ADP 密钥
- ADP 不保存业务真状态
- 后端代理 SSE
- 高数作为首个演示学科
- 变量与章节命名保留扩科能力

额外要求：

- 中文输出
- 优先使用已有对象名，不改名
- 表格和小段说明为主
- 不写营销文案
- 不把知识上传写成“自动立即生效”
