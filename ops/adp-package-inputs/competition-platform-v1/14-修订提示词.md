# 提示词｜生成结果修订

下面我会给你一份已经生成出来的 ADP 应用草案、Agent 草案或平台集成合同草案。  
请你不要重写成另一套新方案，而是在尽量保留原有结构和命名的前提下，做一次“定向修订”。

请重点检查并修复下面 4 类常见偏差：

1. 结果看起来像聊天机器人，而不是 AI 主导学习生命周期平台
2. 把 `LearningMap`、`RerouteEvent`、`KnowledgeRelease` 等业务真状态塞进长期记忆
3. 把新资料写成“上传即生效”，没有 `candidate -> validation -> release/rollback`
4. Agent 分工不清，尤其把多个核心职责重新塞回单个主控

如果你发现其他高风险问题，也可以补充，但不要偏离以上 4 类主修项。

请按下面格式输出：

## 1. 问题清单

逐条列出：

- 问题编号
- 问题位置
- 为什么和当前产品主线冲突
- 风险等级（高 / 中 / 低）

## 2. 修订原则

用 3 到 6 条说明你这次修订坚持的原则，例如：

- 保留既有对象名
- 保留既有 Agent 名
- 只收窄越权职责
- 只补齐缺失门禁与状态边界

## 3. 修订后的内容

只输出被修订的段落或表格，不要把整份长文完全重写一遍。  
每段修订前先标注它对应原稿的哪一部分。

## 4. 最终复核

最后用一张简短清单回答：

- 是否仍然是 `Multi-Agent`
- 是否仍然是 `工作流编排`
- 是否完整覆盖学生主链
- 是否完整覆盖资料入库与策略后台
- 是否保留高数示范但兼容扩科

额外要求：

- 输出语言固定中文
- 不要引入新的大型范围
- 不要把展示站、构建命令、GitHub Pages 写进修订结果
