# 腾讯 ADP 应用包生成输入包使用说明

> 目标：把比赛作品文档压缩成一套适合腾讯“应用包生成助手”消费的输入资产，稳定产出 `ADP 应用核心草案 + 平台集成合同草案 + Agent 提示词草案`。  
> 当前口径：`完整平台版`、`高数优先兼容扩科`。  
> 事实边界：本目录是“生成输入资产”，不是平台控制台真实配置结果，也不代替后续真实导入、真实发布和真实联调证据。

## 1. 资产位置

本目录固定包含 10 份文件：

| 文件 | 用途 |
| --- | --- |
| `A-产品主线包.md` | 锁定产品定位、主闭环、页面范围、MVP 与非范围 |
| `B-Agent与工作流包.md` | 锁定 Agent 拆分、主工作流、资料入库工作流、页面映射 |
| `C-接口契约包.md` | 锁定核心对象、状态边界、ADP 与后端职责分界 |
| `D-教学提示词风格包.md` | 锁定教学风格、共享槽位和高数示范口径 |
| `10-主提示词-应用包骨架生成.md` | 第一轮：生成应用包骨架 |
| `11-Agent批量定义提示词.md` | 第二轮前置：批量生成 Agent 定义表 |
| `12-单Agent细化提示词模板.md` | 第二轮：逐 Agent 生成系统提示词 |
| `13-平台集成合同提示词.md` | 第三轮：生成平台集成合同 |
| `14-修订提示词.md` | 回炉修正生成偏差 |

## 2. 真源材料

这套输入资产只以 6 份文档为业务真源：

1. `doc/作品文档/00-作品总览与阅读地图.md`
2. `doc/作品文档/01-PRD与需求分析.md`
3. `doc/作品文档/05-算法与知识库设计.md`
4. `doc/作品文档/06-接口与API说明.md`
5. `doc/作品文档/11-开发技术文档.md`
6. `doc/开发文档/06-腾讯智能体开发平台集成设计.md`

补充参考只允许来自：

- `ops/adp-prompts/highmath-v1/`：只提供提示词风格与共享契约
- `doc/智能体文档/学科层/高等数学-Agent提示词模板与分层教学规范.md`：只提供教学风格参考

不直接喂给生成助手的材料：

- 页面与交互设计
- 答辩 PPT 大纲
- 演示视频脚本
- 访问与评测手册
- 展示站技术栈、GitHub Pages、Vite 构建命令

## 3. 推荐使用顺序

### 第一步：先生成应用包骨架

1. 打开 `10-主提示词-应用包骨架生成.md`
2. 先把这份提示词发给腾讯助手
3. 再依次粘贴 `A`、`B`、`C`、`D` 四个输入包
4. 等腾讯助手输出：
   - 应用定位
   - Agent 列表
   - 学生主工作流
   - 资料入库工作流
   - 变量清单
   - 长期记忆边界
   - 工具与知识库使用原则

这一轮不要让它直接生成长提示词。

### 第二步：先批量定义，再逐 Agent 细化

1. 用 `11-Agent批量定义提示词.md` 先让它产出 Agent 职责表
2. 再使用 `12-单Agent细化提示词模板.md`
3. 每次只生成一个 Agent
4. 建议顺序：
   - `StarterAgent`
   - `DiagnosisAgent`
   - `MapPlannerAgent`
   - `TutorAgent`
   - `EvaluatorAgent`
   - `ProfileAgent`
   - `NoteSynthAgent`
   - `IngestionAgent`
   - `StrategyAgent`
5. 若腾讯模板强依赖主控 Agent，再补一个轻量 `TeacherOrchestrator`

### 第三步：生成平台集成合同

使用 `13-平台集成合同提示词.md`，让腾讯助手补齐：

- `VisitorId`
- `ConversationId`
- `SYS.Memory`
- `custom_variables`
- 工作流节点输入输出
- SSE 事件口径
- 知识治理状态流
- 审计与回滚约束

### 第四步：回炉修正

拿腾讯助手生成的草案，使用 `14-修订提示词.md` 回炉修正，重点抓 4 类常见偏差：

1. 像聊天机器人
2. 把真状态塞进记忆
3. 把知识上传即生效
4. Agent 分工不清

## 4. 固定默认约束

每轮生成都以这些约束为最高优先级：

- 应用模式：`Multi-Agent`
- 协同方式：`工作流编排`
- 学生主链：`选科 -> 初始地图 -> 短诊断 -> 当前关卡教学 -> 作答评分 -> 地图重规划 -> 画像更新 -> 笔记生成`
- 新资料链路：`上传 -> SaveDoc/OCR -> 声明抽取 -> candidate -> validation -> release/rollback -> 影响域重算`
- 长期记忆只存：偏好、薄弱点、错误模式、节奏特征
- 长期记忆不存：地图真状态、节点完成状态、知识发布状态
- 演示首学科：`高等数学`
- 变量与章节命名：高数优先，但保留扩科能力
- 输出语言：中文
- 风格约束：不是百科问答，不是营销首页，不是假装已实现的全栈成品

## 5. 最终验收清单

用这套清单复核腾讯助手输出：

- 6 个页面都有对应 Agent 或工作流节点承接
- 主链完整覆盖地图、诊断、闯关、评分、反馈、画像、笔记
- 后台完整覆盖资料入库、候选区、校验、发布/回滚、策略快照、审计
- `LearningMap`、`RerouteEvent`、`KnowledgeRelease` 不只存在于 ADP 记忆
- 至少出现高数示范口径，例如“极限/导数”
- 变量命名没有被锁死成只能高数
- 没有误生成营销首页、泛问答助手或自由转交主链
- 评分类 Agent 有结构化输出，不只返回自然语言

## 6. 一句提醒

这套资产的作用是“帮你更稳地喂腾讯助手”，不是替代你做最终平台配置。  
真正进入控制台配置、知识库接入和发布检查时，仍然要遵守当前仓库的 ADP 协作前置：你先登录，我再接手浏览器。
