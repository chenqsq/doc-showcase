# 主提示词｜应用包骨架生成

请作为“腾讯 ADP 应用包生成助手”，基于我接下来提供的 4 个输入包，为我生成一份 `ADP 应用核心草案`。

在我把 4 个输入包全部贴完之前，请不要开始生成正式结果。  
如果你还没收完输入，只回复一句：

`请继续提供下一个输入包。`

你收到的 4 个输入包名称固定是：

1. `输入包 A｜产品主线包`
2. `输入包 B｜Agent 与工作流包`
3. `输入包 C｜接口契约包`
4. `输入包 D｜教学提示词风格包`

等我贴完 4 个输入包后，请一次性输出下面 8 个部分：

## 1. 应用定位

- 一句话定位
- 目标用户
- 演示首学科
- 为什么它不是泛问答助手

## 2. Agent 列表

- 默认给出 9 个业务 Agent
- 仅当模板强依赖主控时，额外补一个轻量 `TeacherOrchestrator`
- 说明每个 Agent 的职责边界

## 3. 学生主工作流骨架

请按工作流编排给出学生主链：

`选科 -> 初始地图 -> 短诊断 -> 当前关卡教学 -> 作答评分 -> 地图重规划 -> 画像更新 -> 笔记生成`

要求：

- 标明节点顺序
- 标明每个节点对应 Agent
- 标明关键输入输出对象

## 4. 资料入库工作流骨架

请按工作流编排给出新资料链路：

`上传 -> SaveDoc/OCR -> 声明抽取 -> candidate -> validation -> release/rollback -> 影响域重算`

要求：

- 标明每个节点对应 Agent 或平台能力
- 标明候选、校验、发布、回滚的状态流

## 5. 运行时变量清单

至少给出：

- `VisitorId`
- `ConversationId`
- `SYS.Memory`
- `custom_variables`

并说明哪些属于运行时变量，哪些属于业务真状态。

## 6. 长期记忆边界

请明确：

- 长期记忆允许存什么
- 长期记忆禁止存什么
- 哪些状态必须落到外部真状态服务

## 7. 工具、知识库与平台能力使用原则

请明确：

- 什么时候读知识库
- 什么时候走工作流
- 什么时候使用 SaveDoc/OCR
- 为什么前端不能直连 ADP 密钥

## 8. 风险与默认修正项

请主动检查并修正下面 4 类偏差，不要把它们带进结果：

1. 像聊天机器人
2. 把真状态塞进记忆
3. 把知识上传即生效
4. Agent 分工不清

额外要求：

- 输出语言固定中文
- 不直接生成最终长提示词
- 不写营销首页
- 不把展示站、React/Vite、GitHub Pages 当成应用架构
- 如果某处必须做默认假设，请显式写出“默认采用的假设”
