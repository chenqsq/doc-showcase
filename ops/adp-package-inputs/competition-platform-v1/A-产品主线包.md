# 输入包 A｜产品主线包

你现在接收的是“产品主线包”。  
请严格按下面内容理解产品，不要把它生成为泛问答机器人、营销首页或“会聊天的 AI 教师”。

## 1. 一句话定位

这是一个由 AI 主动接管学生学习生命周期、并能在新资料进入后继续演化的自学智能体平台。

它不是“学生提问，AI 回答”的问答页，而是：

- 先生成学习地图
- 再做短诊断校准起点
- 安排当前关卡与下一步任务
- 在卡点时插入补桥或复习节点
- 记录成长反馈和画像变化
- 沉淀笔记、思维导图和复习计划
- 让新资料进入知识治理链路后反向影响后续学习路径

## 2. 唯一主闭环

比赛主链固定为：

`选科 -> 开始学习 -> 初始学习地图 -> 短诊断 -> 当前关卡闯关 -> 卡点触发补桥 -> 通关反馈 -> 画像更新 -> 思维导图/结构化笔记 -> 新资料入库后地图变化`

生成应用包时必须围绕这条主闭环，不得继续发散成通用陪聊、通用题库或营销展示产品。

## 3. 固定页面范围

产品固定包含 6 个核心页面：

| 页面 | 角色 | 页面目标 |
| --- | --- | --- |
| 选科与开学页 | 学生 | 一键进入 AI 接管学习 |
| AI 学习地图页 | 学生 | 看到当前所处位置、下一步和为什么这样安排 |
| AI 闯关学习页 | 学生 | 完成讲解、作答、评分、反馈和推进 |
| 笔记复习与成长页 | 学生 | 查看笔记、导图、复习任务和成长变化 |
| 资料注入与知识库演化后台 | 平台管理者 | 上传资料、查看候选知识、发布和回滚 |
| 系统自治与策略分析后台 | 平台管理者 | 查看 Agent 状态、策略快照、异常和审计 |

## 4. MVP / P0 必做范围

以下能力必须体现在应用包骨架里：

- 学生能从选科页一键进入 AI 接管学习
- 平台能生成初始学习地图，并在短诊断后重排第一版主线
- 学生能进入当前关卡，完成一轮讲解、作答、评分与推进
- 系统能在卡点时插入补桥支线，并解释为什么改路线
- 学生能看到画像变化、成长反馈和至少一种复习资产
- 平台管理者能上传新资料，并看到知识演化记录与地图影响

## 5. 平台边界

- 首个演示学科：`高等数学`
- 设计口径：高数优先，但对象命名、变量命名和章节结构保留扩科能力
- 部署口径：这是 ADP 应用与平台集成草案，不是假装完整上线的全栈成品
- 真实业务真状态由外部服务承接，不全部塞进 ADP

## 6. 明确非范围

请不要生成下面这些方向：

- 营销型首页
- 以排行榜、积分榜为主叙事的产品
- 重皮肤化、重卡通化的教育游戏外壳
- “自动训练底层大模型”作为核心卖点
- 多学科全部做满的超大范围设计
- 复杂微服务与大规模运维体系
- 纯自由转交、无结构化工作流的主链

## 7. 生成输出时必须满足的表达要求

- 输出语言固定中文
- 明确说明它不是泛问答助手
- 首屏叙事必须落在学生主链，不做营销介绍
- 任何平台能力描述都要服务于学习生命周期闭环
- 任何知识更新描述都必须经过候选、校验、发布或回滚
